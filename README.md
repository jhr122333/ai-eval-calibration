# AI 평가 캘리브레이션 대시보드

1차 평가와 2차 검수 결과를 비교해 정확도와 수정 패턴을 파악하는 대시보드입니다.

---

## v2.0

### 변경 사항

- **데이터 구조 전환**: 단일 점수 기반에서 `1차 평가(primary)`와 `2차 검수(secondary)` 비교 구조로 변경
- **개요 재설계**: 표준편차/불일치 중심에서 `전체 정확도`, `변경 건수`, `가장 많이 바뀐 항목`, `변경이 많은 콘텐츠` 중심으로 개편
- **평가자 분석 재설계**: 점수 분포/편향 대신 `평가자별 정확도`, `수정이 많은 평가자`, `항목별 변경률`, `변경 사례 보기` 중심으로 개편
- **콘텐츠 분석 재설계**: 표준편차 기반 히트맵을 제거하고 `콘텐츠 × 항목 변경률`과 상세 수정 사례 중심으로 변경
- **변경 사례 드릴다운 추가**: 실제로 어떤 콘텐츠와 항목이 `상향 수정 / 하향 수정` 되었는지 테이블에서 바로 확인 가능
- **데모 데이터 갱신**: 1차 평가자와 검수자, 항목별 수정 패턴이 반영된 새 비교형 데모 데이터셋으로 교체

### v2 주요 화면

- **개요**: 총 갯수, 평가 항목, 전체 정확도, 가장 많이 바뀐 항목
- **평가자 분석**: 평가자별 정확도 차트, 수정이 많은 평가자 랭킹, 항목별 변경률, 변경 사례 필터/테이블
- **콘텐츠 분석**: 콘텐츠 × 항목 변경률, 콘텐츠별 상세 수정 이력
- **평가 가이드라인**: SOP 문서 인앱 열람

### v2 데이터 형식

업로드할 CSV / Excel 파일에는 아래 칼럼이 필요합니다.

```csv
content_id, primary_evaluator_id, reviewer_id, criterion, primary_score, secondary_score, primary_reason, secondary_reason
```

- `primary_score` — 1차 평가 점수, 0/1/2 중 하나
- `secondary_score` — 2차 검수 점수, 0/1/2 중 하나
- `primary_reason`, `secondary_reason` — 필요 시 사유 입력

### v2 핵심 지표

- **전체 정확도** = `primary_score === secondary_score` 비율
- **변경 건수** = 2차 검수에서 점수가 바뀐 항목 수
- **상향 수정** = 2차 점수가 1차보다 높아진 경우
- **하향 수정** = 2차 점수가 1차보다 낮아진 경우

## Power BI 대시보드 버전

> AI 생성 콘텐츠 품질 평가팀의 평가자 간 정확도를 모니터링하는 Power BI 대시보드

### 프로젝트 개요

AI가 제품 SDP(Standard Design Package) 문서 내 텍스트를 바탕으로 생성한 Q&A를 평가자가 채점하고, 검수자가 검토하는 평가 워크플로우에서 **평가 품질을 데이터 기반으로 관리**하기 위해 제작한 대시보드입니다.

기존에는 엑셀로 평가 결과를 관리하며 개인별 정확도를 수작업으로 집계했으나, 이를 Power BI로 자동화하여 **실시간 KPI 모니터링 및 패턴 분석**이 가능하도록 했습니다.

### 대시보드 구성

**Page 1 — 평가 현황 요약**

| 시각화 | 설명 |
|---|---|
| KPI 카드 (4개) | 총 태스크 수 / 링크 불가 수 / 팀 평균 정확도 / 총 변경 수 |
| 평가자별 정확도 테이블 | 12명 평가자의 정확도, 링크 불가 수, 담당 검수자 |
| 기준별 정확도 막대 차트 | 정확성 / 간결성 / 정밀성 항목별 정확도 비교 |

**Page 2 — 월별 추이 & 검수자 분석**

| 시각화 | 설명 |
|---|---|
| 월별 정확도 추이 꺾은선 | 평가자별 6개월 정확도 변화 (2024-10 ~ 2025-03) |
| 검수자별 변경률 막대 차트 | 검수자 3명의 점수 변경 비율 비교 |
| 슬라이서 3개 | 평가자 / 기준 / 기간 필터 (전 페이지 연동) |

### 핵심 지표 정의

**정확도 (Accuracy Rate)**
```
정확도 (%) =
  검수자가 변경하지 않은 평가 수
  ÷ 링크 불가를 제외한 전체 평가 수
  × 100
```
> 링크 불가 항목은 평가 자체가 불가능하므로 분모에서 제외

**변경률 (Change Rate)**
```
변경률 (%) =
  검수자가 점수를 변경한 평가 수
  ÷ 링크 불가를 제외한 전체 평가 수
  × 100
```

### 데이터 스펙

| 항목 | 내용 |
|---|---|
| 평가자 | 12명 (evaluator_01 ~ evaluator_12) |
| 검수자 | 3명 (reviewer_01 ~ reviewer_03) |
| 평가 태스크 | 3,000개 |
| 평가 기준 | 정확성 / 간결성 / 정밀성 (3점 척도: 0, 1, 2) |
| 총 행수 | 9,000행 (3,000 × 3기준) |
| 평가 기간 | 2024년 10월 ~ 2025년 3월 |

**주요 컬럼**

| 컬럼명 | 타입 | 설명 |
|---|---|---|
| task_id | string | 평가 태스크 ID |
| evaluator_id | string | 평가자 ID |
| reviewer_id | string | 검수자 ID |
| criterion | string | 평가 기준 |
| evaluator_score | int | 평가자 점수 (0/1/2) |
| reviewer_score | int | 검수자 변경 점수 (변경 없으면 null) |
| is_changed | boolean | 검수자 점수 변경 여부 |
| link_unavailable | boolean | 링크 불가 여부 |
| eval_date | date | 평가 일자 |
| product_category | string | 제품 카테고리 |

### 주요 인사이트 (데모 데이터 기준)

- **팀 평균 정확도 73.4%** — 평가자 간 편차가 크며 (59% ~ 89%), 하위 그룹 집중 캘리브레이션 필요
- **정밀성 정확도 55.3%** — 3개 기준 중 가장 낮아 평가 기준 재정의 검토 필요
- **링크 불가 585건 (6.5%)** — evaluator_07, 11에 집중 (각 114건, 117건)
- **reviewer_02 변경률 최고** — 담당 평가자(05, 06, 07, 08) 그룹의 재교육 우선 검토

### 사용 기술

- **Power BI Desktop** — 대시보드 시각화
- **DAX** — 정확도, 변경률 측정값 계산
- **Python (pandas, numpy)** — 데모 데이터 생성
- **Power Query** — 데이터 타입 변환 및 전처리

### 실행 방법

1. `data/eval_data.csv` 를 Power BI Desktop에서 불러오기
2. Power Query에서 컬럼 타입 설정 (is_changed, link_unavailable → True/False, eval_date → 날짜)
3. DAX 측정값 5개 생성 (README 내 정의 참고)
4. 시각화 배치 및 슬라이서 페이지 간 동기화
